Introduction

What is
CivicConnect?

CivicConnect is an Android-based citizen complaint management system that enables residents to directly report civic issues — from potholes to power outages — with real-time tracking and administrative resolution workflows.

  • Bridges citizens & local authorities digitally
  • Eliminates manual paperwork & in-person visits
  • Promotes civic accountability and transparency
📱

Citizen-First Design

Anyone can report an issue with their smartphone in under 60 seconds.

🏛️

Why This Was Needed

India processes millions of civic complaints yearly — most delayed due to inefficient, paper-based systems.

SupabaseIssueRepository.java — SELECT + JOIN 
// Fetch all issues for a user, newest first
GET /rest/v1/issues?select=
  *,profiles(username,avatar_url)
  &user_id=eq.{userId}
  &order=created_at.desc

// Equivalent SQL executed by PostgREST:
SELECT i.*, p.username, p.avatar_url
FROM   issues i
JOIN   profiles p ON i.user_id = p.id
WHERE  i.user_id = 'uuid-user-123'
ORDER BY i.created_at DESC;
IssueDetailActivity.java — UPDATE status 
// Admin marks complaint as RESOLVED
PATCH /rest/v1/issues?id=eq.{issueId}
Body: { "status": "RESOLVED",
        "resolved_at": "<ISO timestamp>" }

// Leaderboard — top reporters by resolved count
SELECT p.username, p.avatar_url,
       COUNT(i.id) AS resolved_count
FROM   issues i
JOIN   profiles p ON i.user_id = p.id
WHERE  i.status = 'RESOLVED'
GROUP BY p.id, p.username, p.avatar_url
ORDER BY resolved_count DESC
LIMIT  10;

Exception Handling — 1 / 2

Custom
Exception Classes

Instead of catching a generic Exception everywhere, CivicConnect defines three domain-specific exception classes — each lives in the utils/ package.

🔐

AuthException

utils/AuthException.java

Thrown on wrong credentials, expired JWT, or unauthorized route access.

🌐

NetworkException

utils/NetworkException.java

Wraps HTTP errors (4xx/5xx) and carries the statusCode for targeted retry logic.

🗄️

StorageException

utils/StorageException.java

Raised when photo upload to Supabase Storage fails — file too large, bad format, or bucket full.

NetworkException.java 
public class NetworkException
    extends Exception {

  private final int statusCode;

  public NetworkException(
      String msg, int code) {
    super(msg);
    this.statusCode = code;
  }

  public int getStatusCode() {
    return statusCode;  // e.g. 401, 404, 500
  }
}

LoginActivity.java — multi-catch on sign-in 
try {
  supabase.signIn(email, password,
    new Callback<String>() {
      @Override public void onSuccess(String token) {
        UserSession.save(this, token);
        startActivity(new Intent(
          this, HomeActivity.class));
      }
      @Override public void onError(Exception e) {
        if (e instanceof AuthException) {
          showToast("Invalid email or password");
        } else if (e instanceof NetworkException) {
          NetworkException ne = (NetworkException) e;
          if (ne.getStatusCode() == 503)
            showToast("Server unavailable");
          else
            showToast("No internet connection");
        }
      }
    });
} catch (Exception e) {
  Log.e("LoginActivity", e.getMessage());
}

Try-Catch
use cases

Graceful error recovery for API failures, validation errors, and network issues.

SupabaseHelper.java — submitComplaint() 
public void submitComplaint(
    Complaint c, OnCompleteListener cb) {
 
  try {
    // Validate before sending to DB
    if (c.getDescription().isEmpty()) {
      throw new IllegalArgumentException(
          "Description cannot be empty");
    }
 
    supabase.from("complaints")
            .insert(c)
            .execute();
 
    cb.onSuccess("Complaint submitted!");
 
  } catch (IllegalArgumentException e) {
    cb.onFailure("Validation: " + e.getMessage());
 
  } catch (IOException e) {
    cb.onFailure("No internet connection");
 
  } catch (Exception e) {
    Log.e("CivicConnect", "Error", e);
    cb.onFailure("Unexpected error occurred");
  }
}
ImageUploadHelper.java — uploadImage() 
public void uploadImage(Uri uri,
        String complaintId) {
  try {
    Bitmap bmp = MediaStore.Images.Media
        .getBitmap(getContentResolver(), uri);
 
    ByteArrayOutputStream baos =
        new ByteArrayOutputStream();
    bmp.compress(Bitmap.CompressFormat.JPEG,
        70, baos);
 
    supabase.storage()
        .from("complaint-images")
        .upload(complaintId + ".jpg",
               baos.toByteArray());
 
  } catch (FileNotFoundException e) {
    showToast("Image file not found");
 
  } catch (IOException e) {
    showToast("Upload failed — try again");
  }
}
⚠️
IllegalArgumentException — catches empty form fields before hitting the network
📡
IOException — handles offline state and Supabase upload timeout
🗂️
FileNotFoundException — user picks a deleted image, app recovers gracefully

SupabaseIssueRepository.java — implements IssueRepository 
public class SupabaseIssueRepository
    implements IssueRepository {

  private final SupabaseService service;

  public SupabaseIssueRepository(
      SupabaseService svc) {
    this.service = svc;
  }

  @Override
  public void createIssue(
      Issue issue, Callback<Issue> cb) {
    service.post("/issues",
      JsonParser.toJson(issue), cb);
  }

  @Override
  public void getIssuesByUser(
      String uid, Callback<List<Issue>> cb) {
    service.get("/issues?user_id=eq." + uid, cb);
  }
}

OOP Concepts — 2 / 2

Inheritance &
Polymorphism

Activities extend AppCompatActivity (Inheritance). The repository field is typed as the interface, not the concrete class — classic Polymorphism.

🏛️
Inheritance — HomeActivity, LoginActivity, ReportIssueActivity all extend AppCompatActivity and inherit onCreate, onResume, onPause.
🔀
Polymorphism — the IssueRepository reference in HomeActivity can be pointed to any implementation (Supabase, mock, local) without changing the Activity code.
⬆️
@Override on both createIssue and getIssuesByUser — compiler enforces the contract is fully implemented.
